# System Module (SYS) Frontend

이 디렉토리는 시스템 공통 모듈(SYS)의 프론트엔드 화면과 리소스를 포함합니다.

## 📌 화면 네이밍 룰 (Naming Convention)

프론트엔드 화면(Vue 파일)은 **대분류 업무 코드**와 **화면 번호(목적)**를 조합하여 명명합니다.

- **디렉토리:** 대분류 업무 코드 (예: `SYST01` - 사용자 관리, `SYST02` - 권한 관리)
- **파일 네이밍:** 대분류 업무 코드 + `V` (View) + 일련번호 3자리 (예: `SYST01V001.vue`)
  - `V001`: 주로 목록 조회 (List)
  - `V002`: 주로 상세 조회 및 등록/수정 (Detail/Form)

### 6. 테이블 명세서 관리 (SYST06)

데이터베이스 테이블, 컬럼, 인덱스 명세를 관리하고 변경 이력을 추적하며 SQL 스크립트를 생성합니다.

| 폴더 위치 | 화면명             | 파일명           | 비고                                                                        |
| :-------- | :----------------- | :--------------- | :-------------------------------------------------------------------------- |
| `SYST06/` | 테이블 명세서 조회 | `SYST06V001.vue` | 시스템 전체 테이블 및 컬럼, 인덱스 목록 조회                                |
| `SYST06/` | 테이블 명세서 상세 | `SYST06V002.vue` | 테이블, 필드, 인덱스 상세 수정, 변경 로그 확인, SQL 스크립트 실행 버튼 포함 |

**[SYST06 화면 설계 상세]**

- **SYST06V001 (목록 화면)**
  - **검색 영역:** 모듈명(`module`), 테이블 물리명(`tablen`), 테이블 논리명(`tableNm`) 검색 조건 제공.
  - **데이터 그리드:** `sysTable`과 `sysTablex`를 조인하여 테이블 기본 정보 목록 표시.
  - **액션:** 신규 등록 버튼, 그리드 행 클릭 시 `SYST06V002` 상세 화면으로 이동.
- **SYST06V002 (상세/수정 화면)**
  - **상단 (마스터 영역):** 테이블 기본 정보 (`tablen`, `module`, `tableNm`, `description`) 입력 폼.
  - **중단 (디테일 탭/그리드):**
    1.  **필드(컬럼) 명세:** `sysFields`, `sysFieldsx` 목록. (필드명, 타입, 길이, NULL여부, PK여부, 자동증가(A_I) 여부, 순서 재배치 등 관리)
    2.  **인덱스 명세:** `sysTableIndex`, `sysIndexFields` 목록. (인덱스명, 대상 필드, Unique 여부, 순서 재배치 등 관리)
    3.  **변경 이력 (History):** `sysTableHistory` 테이블을 조회하여 대상 테이블/필드/인덱스의 변경 내역(Before/After) 표시.
  - **액션 버튼:**
    - **저장 (Save):** 변경된 테이블/필드/인덱스 정보를 DB에 저장하고, 이전 데이터와 비교하여 변경된 속성이 있다면 `sysTableHistory`에 이력을 자동 기록.
    - **SQL 스크립트 생성 (Generate SQL):** 저장된 명세 및 이력을 바탕으로 실제 DB에 적용 가능한 `CREATE TABLE`, `ALTER TABLE`, `CREATE INDEX` 등의 DDL 구문을 생성하여 모달로 제공/실행.

## 🛣️ 라우터 매핑 구조 (Routing Structure)

| 경로 (Path)           | 파라미터            | 매핑 컴포넌트     | 설명                            |
| :-------------------- | :------------------ | :---------------- | :------------------------------ |
| `/sys/syst00`         | -                   | `SYST00V001.vue`  | 시스템 대시보드                 |
| `/sys/syst01`         | -                   | `SYST01V001.vue`  | 사용자 관리 목록                |

| `/sys/syst03`         | -                   | `SYST03V001.vue`  | 메뉴 관리 목록                  |

## 🌐 API 통신 모듈화 (API Structure)

**[제약 사항 강제]** Vue 컴포넌트 내에서 `fetch()` 또는 `axios`를 직접 사용하는 것은 엄격하게 금지됩니다. 
모든 백엔드 API 통신 로직은 반드시 각 모듈 내의 `api/` 디렉토리에 도메인별로 분리하여 관리하고, 컴포넌트에서는 생성된 Axios API 모듈을 임포트(`import`)하여 호출해야만 합니다.

### 시스템 모듈 API 구조 (`src/frontend/module_sys/api/`)
- `userApi.js`: `/api/sys/users` 엔드포인트 연동 (사용자 관리용)
- `roleApi.js`: `/api/sys/roles` 엔드포인트 연동 (권한 관리용)
- `menuApi.js`: `/api/sys/menus` 엔드포인트 연동 (메뉴 관리용)
- `codeApi.js`: `/api/sys/codes` 엔드포인트 연동 (공통 코드 관리용)
- `configApi.js`: `/api/sys/configs` 엔드포인트 연동 (시스템 설정용)
- `tableSpecApi.js`: `/api/sys/tablespecs` 엔드포인트 연동 (테이블 명세서 관리용)
