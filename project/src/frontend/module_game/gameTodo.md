  - 두 국가의 전사(Warrior)들이 국경선으로 집결하여 집단 전투(`ATTACKING`)를 벌이는 로직 개발.

  - 반대로 자원이 0이 되어 멸망해가는 마을의 구성원들이 주변의 부유한 마을로 흡수되는 알고리즘.
- [ ] **상인(TRADER)과 길(Road) 인프라 형성**:
  - 자원이 쏠려있는 두 마을 간 교역을 담당하는 상인 직업 추가.
  - 상인이 자주 왕복하는 좌표를 기록하여 땅의 색깔이 흙길(Road)처럼 변하며(Canvas Pixel 조작), 해당 길 위에서는 이동 속도(`speed`)가 1.5배 증가하는 인프라 시스템.

## 3. 🚀 엔진 최적화 및 코어 모듈화 (Optimization & Modularization)
현재의 QuadTree와 Offscreen Canvas도 훌륭하지만, 1만 개체 이상을 목표로 할 경우 추가 조치가 필요합니다.

### ✅ 3.1 Web Worker를 이용한 멀티스레딩 (Completed)
- **내용**: 브라우저 메인 스레드는 `requestAnimationFrame` 렌더링만 담당하며, `World.js`의 물리/AI 로직은 Web Worker 스레드에서 완전히 분리되어 병렬 처리됩니다.
- **구현 방식**: 상태 동기화(State Sync) 방식(방안 1)을 채택하여 Worker에서 계산된 객체 스냅샷(`exportState`)을 `postMessage`로 전달하고, Main 스레드는 `importState`를 통해 뷰포트 컬링과 Canvas Rendering만 수행합니다.

### 3.2 Viewport Culling 고도화 (Chunking)
- **내용**: 현재 화면에 보이는 범위만 렌더링하는 것을 넘어, 전체 3200x3200 맵 자체를 여러 개의 `Chunk` 단위 오프스크린 캔버스로 쪼개어 관리. 카메라가 이동할 때 해당 Chunk만 메인 화면에 그리도록 전환.

### 3.3 파티클(Particle) 효과 도입
- **내용**: 벌목 시 나뭇잎 흩날림, 전투 시 핏방울, 건설 시 먼지 효과 등 비주얼 폴리싱(Visual Polish).
- **주의점**: 시각 효과 배열 역시 철저한 오브젝트 풀링(Object Pooling)을 적용하여 프레임 드랍을 원천 차단해야 함.

### 3.4 코어 엔진(World.js)의 시스템(System) 분할 (Decoupling God Object)
- **내용**: 현재 `World.js` 한 곳에 묶여 있는 렌더링, 시간, 재해, 외교 연산 등을 독립된 컨트롤러 객체로 분할하여 엔진의 유지보수성 극대화.
- **진행 목표**:

  - ~~**`DisasterSystem.js`**: 지진 타이머, 토네이도 스폰, 가뭄 등의 재난 이벤트 및 데미지 연산 분리.~~ (완료)
  - **`Renderer.js`**: Z-index 정렬, 배경 캔버스 복사, 카메라 오프셋 적용 등 순수 렌더링 파이프라인 전담 분리.

## 4. 🎭 게임 요소 및 상호작용 풍부화 (Gameplay Enrichment)
- [ ] **유전자 및 특성(Traits) 시스템 도입**:
  - 지성체 및 동물들이 태어날 때 무작위 특성(예: '강인함', '게으름', '천재', '불로장생')을 부여받음.
  - 특성에 따라 이동 속도, 작업 효율, 수명, 전투력 등에 보너스/페널티 적용 및 부모 세대의 특성이 자식에게 유전되는 진화 알고리즘.
- [ ] **다양한 생물군계(Biome) 및 타일 지형**:
  - 기존의 단일한 잔디밭 외에 사막(Desert), 늪지(Swamp), 설원(Snow), 용암 지대(Lava) 등의 다채로운 지형(Biome) 생성.
  - 생물군계 타일에 따라 자라나는 식물 종류와 이동 속도 페널티, 서식하는 동물(예: 펭귄, 낙타)이 달라지는 생태계 다양화.
- [ ] **문명의 시대(Age of Civilization) 진화**:
  - 마을의 '지식(Knowledge)' 자원이 특정 수치에 도달하면 문명의 시대(예: 석기 -> 청동기 -> 철기 -> 르네상스)가 발전함.
  - 시대가 발전함에 따라 건물들의 스프라이트가 현대화되고, 전사들의 무기 및 전투력이 상승하는 메커니즘.
- [ ] **전염병(Plague) 메커니즘 및 의사(Doctor)**:
  - 인구 밀집도가 높거나 시체가 방치된 환경일 때 무작위 개체에게 전염병 발생.
  - 감염자는 이동 시 주변에 병원체를 퍼뜨리며 체력이 지속 감소. 마을 내 '의사(Doctor)' 직업이 이를 파악하고 치료제를 투여하는 FSM 추가.
- [ ] **신앙(Religion) 및 이념 시스템**:
  - 마을 주민들이 무작위로 특정 '이념'이나 '신앙'을 가지게 되며, 이를 숭배하는 특수 건물 '기념비(Monument)'를 건설.
  - 종교가 다른 마을 간에는 `Diplomacy` 마찰이 더 쉽게 발생하여 종교 전쟁 발발.
